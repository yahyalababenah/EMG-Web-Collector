# تقرير فني: نظام استحواذ بيانات الإشارات العضلية الكهربائية للذكاء الاصطناعي
(sEMG Data Acquisition System for 1D CNN)

## 1. الهدف الشامل للمشروع (Project Goal)
الهدف الرئيسي من هذا المشروع هو هندسة وتطوير "أداة ويب مدمجة" (Web-based Embedded Tool) لجمع وتسجيل بيانات الإشارات العضلية الكهربائية السطحية (sEMG) بدقة عالية.
يُصمم هذا النظام خصيصاً لبناء قاعدة بيانات (Dataset) لـ 15 متطوعاً، بحيث تكون البيانات الناتجة مهيأة وموسومة (Labeled) تلقائياً وجاهزة لتدريب نموذج ذكاء اصطناعي من نوع 1D CNN. يهدف النموذج النهائي إلى التمييز بين 3 حركات أساسية لليد:

Class 0: وضع الراحة (Rest)

Class 1: الانقباض الكامل / القبض (Full Grip)

Class 2: القرص بأصبعين (Pinch)

يعالج المشروع مشاكل التشويش، وتزامن الحركات، واستخراج الميزات الرياضية (RMS & SSI) في الوقت الفعلي (Real-time)، مما يختصر مئات الساعات من معالجة البيانات لاحقاً.

### 2. متطلبات النظام (System Requirements)
#### أ . المتطلبات المادية (Hardware):

متحكم دقيق: لوحة ESP32-CAM (تُستخدم كمتحكم فقط لمعالجها القوي ومحولها التناظري الدقيق 12-bit ADC).

الحساس: حساس MyoWare Muscle Sensor (أحادي القناة - Single Channel).

أداة البرمجة: مبرمج خارجي FTDI Adapter لرفع الكود ونقل البيانات التسلسلية.

أقطاب طبية (Electrodes): لتوصيل الحساس بالعضلة المستهدفة.

#### ب. المتطلبات البرمجية (Software):

بيئة التطوير: Arduino IDE (لبرمجة القطعة)، و VS Code (لتطوير الويب).

المتصفح: Google Chrome أو Edge (لضرورة دعم واجهة Web Serial API).

مكتبات الويب: Tailwind CSS (للتصميم)، و Chart.js (للرسم البياني اللحظي).

### 3. سير العمل وتدفق البيانات (System Workflow)
النظام يعمل عبر ثلاث طبقات متزامنة:

طبقة العتاد (Hardware Layer): تقوم لوحة الـ ESP32-CAM بقراءة الجهد الكهربائي للعضلة (0 إلى 4095) بمعدل 500 مرة في الثانية (500Hz). يتم إرسال كل قراءة مع الطابع الزمني (Timestamp) عبر منفذ الـ USB إلى جهاز الكمبيوتر.

طبقة الويب والتوجيه (Frontend & Prompter Layer):
يتصل المتصفح بالقطعة عبر تقنية Web Serial API. يعرض الموقع "مُلقّناً بصرياً" (Prompter) يوجه المتطوع. يتغير لون الشاشة للأخضر (إشارة للانقباض) وللأحمر (إشارة للراحة) لمدة 3 ثوانٍ لكل مرحلة. هذا يضمن أن البيانات المسجلة متزامنة بدقة مع الحركة الفعلية للمتطوع.

طبقة المعالجة والتصدير (Processing & Export Layer):
أثناء تدفق البيانات، يقوم كود JavaScript بتجميعها في "نوافذ منزلقة" (Sliding Windows) بحجم 100 عينة (حوالي 200 ملي ثانية). يحسب الكود ميزات الـ RMS والـ SSI لحظياً. عند انتهاء التكرارات الـ 10 لكل حركة، يتم دمج البيانات وتحميلها كملف .csv تلقائياً.

#### 6. شرح تفصيلي لوظيفة الأكواد (Code Modules Explanation)
#### أ. ملف firmware/esp32_logger.ino (المتحكم الدقيق)
الوظيفة: هو البرنامج الذي يُرفع على لوحة ESP32-CAM.

آلية العمل: * يقوم بإيقاف تشغيل الـ WiFi والبلوتوث برمجياً لضمان عدم وجود أي تشويش كهرومغناطيسي على إشارة العضلة التناظرية.

يستخدم دبوس (GPIO 14) التابع لـ ADC1 لقراءة الحساس لضمان استقرار الإشارة.

يستخدم خوارزمية توقيت تعتمد على millis() لضمان إرسال عينة واحدة كل 2 ملي ثانية بشكل صارم، ليحقق معدل نقل ثابت (500Hz).

يرسل البيانات بصيغة نصية: Timestamp,EMG_Value.

#### ب. ملف index.html (هيكل الواجهة)
الوظيفة: بناء واجهة المستخدم الرسومية (GUI).

آلية العمل:

يحتوي على حقول إدخال لـ Subject ID و Session Number لتوثيق البيانات.

يحتوي على أزرار التحكم (الاتصال بالبورت، بدء الجلسة، وتحميل البيانات).

يضم منطقة (Canvas) مخصصة لمكتبة Chart.js لرسم إشارة العضلة للمتطوع بشكل حي.

يحتوي على "صندوق المُلقّن" (Prompter Box) الذي تتغير نصوصه وألوانه لتوجيه المتطوع.

#### ج. ملف style.css (التأثيرات البصرية)
الوظيفة: تحسين التفاعل البصري بين النظام والمستخدم (UX).

آلية العمل:

يوفر تأثير "النبض" (Pulse Animation) باللون الأخضر الصارخ عندما يحين وقت "الانقباض" (Action) لجعل المتطوع يركز على الحركة.

يوفر تأثيراً باللون الأحمر الداكن لوقت "الراحة" (Rest) لضمان استرخاء العضلة تماماً وتجنب تلوث البيانات.

#### د. ملف app.js (العقل المدبر لمعالجة البيانات)
الوظيفة: إدارة الاتصال، تزامن البروتوكول، معالجة الإشارات، والتصدير.

آلية العمل:

Web Serial API: يفتح منفذ الـ USB ويقرأ تدفق البيانات النصية القادمة من الـ ESP32، ويقوم بتنظيفها (Parsing).

Sliding Window & Feature Extraction: يخزن القراءات في مصفوفة مؤقتة (Buffer)، وبمجرد اكتمال 100 قراءة، يطبق معادلات رياضية لحساب الـ Root Mean Square (RMS) والـ Simple Square Integral (SSI).

Auto-Labeling: يمتلك حلقة زمنية (Loop) تدير الجلسة؛ حيث يقوم بإعطاء قيمة 1 لحركة القبض، و 2 للقرص، و 0 للراحة، ويربط هذا الرقم بكل قراءة لحظياً.

CSV Generator: يجمع البيانات في هيكل منظم (Time, Raw, Class, ID, RMS, SSI) ويستخدم كائن الـ Blob لإنشاء ملف بيانات متكامل وتنزيله على جهاز الكمبيوتر كملف CSV نظيف وجاهز.

## الخلاصة:
هذا النظام ينقل عملية جمع الإشارات الحيوية من بيئات سطر الأوامر (Command Line) المعقدة إلى بيئة ويب تفاعلية، مما يضمن جودة بيانات احترافية (Clean Data) تقلل من أخطاء الموديل، وتوفر أساساً متيناً لتطوير الذراع الصناعية الذكية.
